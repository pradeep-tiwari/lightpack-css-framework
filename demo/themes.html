Theme Customization Guide

Learn how to customize Lightpack CSS to match your brand and create custom themes.

Overview

Lightpack CSS uses CSS custom properties (variables) for theming, making it incredibly easy to customize colors, typography, spacing, and more. The framework includes built-in light and dark themes that automatically adapt based on the .theme-dark class.

How Theming Works

CSS Variables Architecture

All design tokens are defined in src/css/_variables.css using CSS custom properties:

:root {
  /* Base colors */
  --white: #ffffff;
  --black: #0a0a0a;
  
  /* Semantic colors */
  --color-background: var(--white);
  --color-text: var(--black);
  --color-primary: var(--black);
  
  /* Typography */
  --font-family: system-ui, -apple-system, ...;
  --fs-base: 1rem;
  --fw-normal: 400;
}

Dark Theme Override

Dark theme is activated by adding .theme-dark class to the body:

.theme-dark {
  --color-background: #0a0a0a;
  --color-text: #f5f5f5;
  --color-primary: #ffffff;
  --color-surface: #1a1a1a;
  /* ... more overrides */
}

Customization Methods

Method 1: Override CSS Variables (Recommended)

Add custom CSS after loading Lightpack:

<link rel="stylesheet" href="lightpack.css">
<style>
  :root {
    /* Override primary color */
    --color-primary: #3b82f6;
    --color-primary-light: #60a5fa;
    --color-primary-dark: #2563eb;
    
    /* Override typography */
    --font-family: 'Inter', sans-serif;
    --fs-base: 1.125rem;
  }
  
  .theme-dark {
    --color-primary: #60a5fa;
    --color-background: #111827;
  }
</style>

Method 2: Fork and Modify Source

Edit src/css/_variables.css directly:

  1. Clone the repository
  2. Modify variables in src/css/_variables.css
  3. Run npm run build
  4. Use your custom build

Method 3: CSS Class Overrides

Create custom theme classes:

.theme-ocean {
  --color-primary: #0ea5e9;
  --color-success: #10b981;
  --color-background: #f0f9ff;
}

.theme-sunset {
  --color-primary: #f97316;
  --color-background: #fff7ed;
}

Colors

  • --color-success
  • --color-danger
  • --color-warning
  • --color-info
  • --color-background
  • --color-text

  • --font-family-sans
  • --font-family-serif

  • --fs-xs to --fs-5xl
  • --fw-thin to --fw-black

  • --g-4 (1.5rem)
  • --g-5 (2rem)
  • --g-6 to --g-9
Effects
  • --radius-sm/md/lg
  • --shadow-sm/md/lg
  • --transition-fast/base/slow
  • --easing-base

Best Practices

  • Always test both light and dark themes
  • Maintain sufficient color contrast for accessibility (WCAG AA: 4.5:1)
  • Use semantic color names (primary, success, danger) instead of specific colors
  • Test your theme with all components
  • Keep font sizes relative (rem/em) for better scalability
  • Document your custom variables for team collaboration
